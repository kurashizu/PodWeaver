# Audience
听众是技术爱好者，他们对 Arch Linux, NixOS 有一定了解，追求系统的"优雅"与"健壮"。这段音频将在深夜睡前播放，所以语气要从容，不要有激烈的转折。

# Style Guidelines (重要)
- **口语化：** 使用"咱们、其实、你想想看、简单来说"等词汇
- **长句子与短句结合：** 模仿人类思考时的停顿，适当加入"嗯..."、"当然..."
- **技术比喻：** 减少生硬的代码陈述，多用物理世界的例子
- **禁忌：** 严禁出现 Markdown 标题（如 # 第一章），请直接用主持人串场词衔接，严禁使用描述背景的语言（如"背景音乐响起"），确保输出的所有内容都是可以被TTS系统直接朗读的纯文本内容。严禁出现代码，命令行提示符，或者任何形式的代码块。
